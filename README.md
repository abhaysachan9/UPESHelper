# UPES Helper — RAG Chatbot

A smart university assistant for UPES built with **plain JavaScript + Node.js**, **Gemini AI**, and **Upstash Vector DB**. No frameworks — just clean, modular code.

---

## ✨ Features

- 🎨 Modern UI cloned from upeshelper.netlify.app (with improvements)
- 🌙 Dark/Light mode + Font size accessibility
- 💬 Chat interface with Markdown rendering
- 🎙️ Voice input (Web Speech API)
- 🤖 Gemini 1.5 Flash for answer generation
- 📚 Upstash Vector DB for RAG knowledge retrieval
- 🕷️ Built-in crawler to index UPES website content
- 💾 Chat history persisted in localStorage

## 🚀 Quick Start

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env` with your credentials:

```env
# Google Gemini API — https://aistudio.google.com/
GEMINI_API_KEY=your_gemini_api_key_here

# Upstash Vector — https://console.upstash.com/vector
# ⚠️ Create an index with embedding model enabled (e.g., "text-embedding-3-small")
UPSTASH_VECTOR_REST_URL=https://your-index.upstash.io
UPSTASH_VECTOR_REST_TOKEN=your_token_here

PORT=3000
CRAWL_BASE_URL=https://www.upes.ac.in
```

### 3. Run the crawler

For static pages:
```bash
npm run crawl
```

For JavaScript-heavy pages (after configuring `scripts/dynamic-pages-config.js`):
```bash
npm run crawl:dynamic
```

Or crawl everything:
```bash
npm run crawl:all
```

This crawls UPES pages and saves them to `crawled-data/`.

### 4. Index content into Upstash

```bash
npm run index
```

This chunks and upserts all crawled content into your Upstash Vector index.

### 5. Start the server

```bash
npm run dev    # dev mode with hot-reload
# or
npm start      # production
```

Visit **http://localhost:3000** 🎉

## 🔑 Getting API Keys

### Gemini API Key
1. Go to [Google AI Studio](https://aistudio.google.com/)
2. Create a new API key
3. Paste into `GEMINI_API_KEY` in `.env`

### Upstash Vector DB
1. Go to [Upstash Console](https://console.upstash.com/vector)
2. Create a new Vector Index
3. **Important**: Enable an embedding model (e.g., `text-embedding-3-small`) so Upstash auto-embeds text
4. Copy the REST URL and token into `.env`

## 🌐 API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/` | Chat UI |
| `POST` | `/api/chat` | Send a question, get RAG answer |
| `GET` | `/api/health` | Health check |

### POST /api/chat

**Request:**
```json
{ "message": "What is the fees structure at UPES?" }
```

**Response:**
```json
{
  "answer": "The fees at UPES vary by program...",
  "sources": ["https://www.upes.ac.in/fees"]
}
```

## 🕷️ Crawler Config

The project supports two types of crawling:

### Static Crawling (Default)
Fast HTML parsing for standard web pages:

```bash
npm run crawl
```

### Dynamic Crawling (JavaScript-heavy pages)
Uses Puppeteer to render JavaScript content:

```bash
npm run crawl:dynamic
```

### Crawl Everything
Run both crawlers:

```bash
npm run crawl:all
```

Configure via environment variables in `.env`:

| Variable | Default | Description |
|----------|---------|-------------|
| `CRAWL_BASE_URL` | `https://www.upes.ac.in` | Starting URL |
| `MAX_PAGES` | `50` | Max pages to crawl |
| `PUPPETEER_HEADLESS` | `true` | Run browser in headless mode |

To add JavaScript-heavy pages for dynamic crawling, edit `scripts/dynamic-pages-config.js`.

See [scripts/CRAWLING.md](scripts/CRAWLING.md) for detailed documentation.

## 🏗️ How RAG Works

```
User Question
     ↓
Upstash Vector Query (semantic search)
     ↓
Top-5 relevant chunks retrieved
     ↓
Gemini builds answer using ONLY those chunks
     ↓
Response + source URLs returned
```

The system is strictly knowledge-base-only — Gemini is instructed not to use outside knowledge, ensuring all answers come from the indexed UPES content.
